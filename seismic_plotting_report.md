# 地震走時與波形剖面圖製作報告

本報告詳述了利用 CWASN 網路 SAC 格式地震資料，製作高品質「走時與波形剖面圖」（Record Section）的完整流程，包含程式編寫、遇到的環境挑戰、除錯過程及視覺設計優化。

## 1. 任務目標
將位於 `20260501123955_CWASN` 資料夾中的 500 多個 SAC 地震波形檔案，轉換為一張專業的地震波剖面圖。要求 X 軸為震央距 (km)，Y 軸為時間 (s)，並參考特定美學風格進行設計。

---

## 2. 程式編寫與邏輯設計
核心開發使用 Python 及其專業地震學庫 **ObsPy** 以及繪圖庫 **Matplotlib**。

### A. 資料讀取與前處理
*   **多檔案讀取**：使用 `obspy.read` 配合萬用字元 (`*Z.D*.SAC`) 一次性加載所有垂直分量資料。
*   **測站去重 (Deduplication)**：原始資料包含多種感應器（如寬頻 HHZ、短週期 EHZ、加速度計 HLZ）。我們設計了**優先權邏輯**（HHZ > EHZ > HLZ），確保每個測站只選取品質最好的一條波形，避免圖面重疊混亂。
*   **截取與濾波**：為提升效能，僅截取發震時間（2026-05-01 12:39:55）前後共 130 秒的資料，並進行 0.5 - 5.0 Hz 的帶通濾波。

### B. 走時圖座標計算
*   **震央距提取**：優先從 SAC Header 的 `dist` 或 `gcarc` 讀取距離；若無，則利用測站座標 (`stla`, `stlo`) 與震央座標 (`evla`, `evlo`) 即時計算。
*   **橫軸合成**：公式為 `X = 震央距 + (正規化振幅 * 放大倍率)`。
*   **縱軸定義**：Y 為 `絕對時間 - 發震時間`，單位為秒。

---

## 3. 遇到的困難與除錯過程 (Debugging)

在執行過程中遇到了數個關鍵錯誤，導致程式一度無法正常生成圖片。

### 錯誤一：套件缺失與環境限制
*   **問題**：初始嘗試在預設環境執行，報錯 `ModuleNotFoundError: No module named 'obspy'`。
*   **除錯**：透過 `conda env list` 偵測到用戶電腦中存在 `py3.9_1` 環境，並確認該環境包含 ObsPy。

### 錯誤二：系統沉默崩潰 (Exit Code 1)
*   **問題**：在 `py3.9_1` 執行時，程式在存檔 `savefig` 階段會無預警中斷（Silent Crash），且無任何報錯訊息。
*   除錯過程：
    1.  **記憶體壓力分析**：500 多條高取樣率 (100Hz) 長波形會產生數千萬個數據點，導致 Matplotlib 繪圖引擎超載。
    2.  **優化策略**：引入 **「降採樣」(Decimation)** 邏輯，將 100Hz 資料降為 10Hz，極大減少了數據點。
    3.  **核心依賴衝突**：發現 `numpy 2.0.2` 在該 Python 3.9 環境中與 Matplotlib 不相容，導致渲染引擎崩潰。最終將 numpy **降級至 1.26.4**，成功解決崩潰問題。

### 錯誤三：資料長度不一引起的崩潰
*   **問題**：部分 SAC 檔案存檔範圍不包含地震發生時刻，`trim` 之後變成 0 點，導致濾波器報錯 `ValueError: cannot reshape array of size 0`。
*   **修復**：加入 `if tr.stats.npts == 0: continue` 的檢查點，自動跳過無效檔案。

---

## 4. 視覺設計與優化
為了符合用戶提供的參考圖（螢幕擷取畫面）風格，我們進行了以下設計：

1.  **多色循環系統**：定義了一組包含深紅、橙色、深綠、藍、紫等 10 種顏色的循環清單，使不同測站的波形清晰可辨。
2.  **測站標註**Ha：在每條波形的上方（-2s 位置）利用 `ax.text` 繪製垂直旋轉的測站名稱，並與波形同色。
3.  **座標軸與標題**：
    *   標題採用 ISO 8601 格式的 UTC 時間。
    *   Y 軸（時間）設定為向上增加（0 底部），符合用戶提供的參考圖習慣。
4.  **專業輸出**：強制使用白色背景與精細的灰色網格，設定 DPI 為 200 以確保圖片清晰度。
